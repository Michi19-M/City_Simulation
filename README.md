# Симулация на град

В този проект се реализира симулация на град с възможност за инициализиране, добавяне на сгради и жители.

---

## Град

Градът представлява матрица, където всеки елемент е сграда.

### Видове сгради

- **Тип:**  
  - Модерни — наем 1000 лв.
  - Панелки — наем 500 лв.
  - Общежитие — наем 40 лв.

- **Локация:**  
  - **Централна** — наемът се умножава по 2.5. Централните сгради са тези, които се намират на разстояние min(n, m)/8 от центъра, където n и m са ширина и дължина на града.
  - **Крайна** — наемът се намалява с 20%. Това са сгради, които са на разстояние по-голямо от 6\*min(n,m)/8.
  
Във всяка сграда могат да живеят произволен брой души, ненадминаващ определен капацитет.

---

## Жител

Всеки жител има:
- Име
- Професия
- Целочислени характеристики:
  - Щастие
  - Пари
  - Точки живот

### Ежедневие

- Ставане
- Плащане на наема (ако е първият ден на месеца)
- Плащане на 50 лв. за храна всеки ден
- Сън

### Професии

- **Учител**  
  - Заплата: 1200–1300 лв./месец  
  - Щастие: расте всеки месец `happiness = min(happiness+1, 100)`
- **Програмист**  
  - Заплата: 2000–5000 лв./месец  
  - Щастие: намалява всеки месец `happiness = max(0, happiness-1)`
- **Миньор**  
  - Заплата: 1000–3000 лв./месец  
  - Точки живот: намаляват всеки месец `life = max(0, life-1)`
- **Безработен**  
  - Заплата: 0 лв./месец  
  - Точки живот: намаляват всеки месец `life = max(0, life-1)`

## Програмен интерфейс (Меню с команди)

### Създаване на град

```
>> generate <n> <m>
```
Създава град с размери n x m, със случайно генерирани сгради и жители.  
**Output:** <сегашната дата на симулацията>

---

### Добавяне на хора

```
>> add <n> <m> <name> <job> <happiness> <money> <life>
```
Добавя жител на позиция (n, m) с дадени характеристики.  
**Output:** Резултат от операцията (успешна/неуспешна)

---

### Премахване на хора

```
>> remove <n> <m> <name>
```
Премахва човек от симулацията.  
**Output:** Резултат от операцията (успешна/неуспешна)

---

### Стъпки в симулацията

```
>> step
```
Хората изживяват един ден. На стандартния изход се извежда броят хора с 0 щастие, 0 живот и 0 пари. Те се премахват от симулацията.  
**Output:** <числена информация>

```
>> step n
```
Хората изживяват n дни. Ако n е отрицателно, симулацията се връща назад.  
**Output:** <числена информация>

---

### Информация за симулацията

```
>> info x y
```
Информация за имената на хората в сградата на локация (x, y) и информация за сграда.

```
>> info x y name
```
Пълна информация за човек с име `name` на локация (x, y), включително история на характеристиките му.

```
>> info
```
Пълна структурирана информация за цялата симулация.

**Output пример:**
```
Location 0 0:
<Тип къща>:
    <Име на човек 1>
        <Информация за човек 1>
    ...
    <Име на човек k_(0,0)>
        <Информация за човек k_(0,0)>
...
Location n m:
<Тип къща>:
    <Име на човек 1>
        <Информация за човек 1>
    ...
    <Име на човек k_(n,m)>
        <Информация за човек k_(n,m)>
```

### Сериализация на симулацията

```

```

```
>> load <name>
```
Зарежда симулация от файл.
